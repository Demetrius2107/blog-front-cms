## 2. 产品概述

### 2.1 产品定位
博客后台管理系统是一个面向个人/团队博客的内容管理平台，提供文章发布、分类标签管理、评论审核、用户管理等核心功能，帮助博主高效管理博客内容。

### 2.2 目标用户
- 个人技术博主
- 团队博客管理员
- 内容运营人员

### 2.3 核心价值
- **高效管理**: 统一管理文章、分类、标签、评论等内容
- **数据可视化**: 通过仪表盘直观展示博客运营数据
- **操作便捷**: 简洁的UI设计，降低学习成本

## 4. 非功能需求

### 4.1 性能要求
- 首屏加载时间 < 3s
- 页面路由切换动画流畅（< 100ms）
- 大列表渲染使用虚拟滚动（> 500条数据时）

### 4.2 兼容性
- Chrome >= 90
- Firefox >= 88
- Edge >= 90
- Safari >= 14

### 4.3 安全性
- Token 存储在 localStorage（生产环境建议 HttpOnly Cookie）
- XSS 防护：前端输出转义
- CSRF 防护：后端配合 Token 校验
- 敏感操作需二次确认

### 4.4 可用性
- 响应式布局适配（最小支持 1280px）
- 侧边栏可折叠
- 操作反馈明确（Loading / Message / MessageBox）

## 6. 接口规范

### 6.1 统一响应格式

```typescript
interface ApiResponse<T = any> {
  code: number        // 200 成功, 其他失败
  message: string     // 提示信息
  data: T             // 业务数据
}
```

### 6.2 接口清单

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/auth/login | 用户登录 |
| POST | /api/auth/logout | 退出登录 |
| GET | /api/auth/userinfo | 获取当前用户信息 |
| GET | /api/article/list | 文章列表（分页） |
| GET | /api/article/:id | 文章详情 |
| POST | /api/article | 创建文章 |
| PUT | /api/article/:id | 更新文章 |
| DELETE | /api/article/:id | 删除文章 |
| GET | /api/category/list | 分类列表 |
| POST | /api/category | 创建分类 |
| PUT | /api/category/:id | 更新分类 |
| DELETE | /api/category/:id | 删除分类 |
| GET | /api/tag/list | 标签列表 |
| POST | /api/tag | 创建标签 |
| PUT | /api/tag/:id | 更新标签 |
| DELETE | /api/tag/:id | 删除标签 |
| GET | /api/comment/list | 评论列表 |
| PUT | /api/comment/:id/approve | 审核通过评论 |
| DELETE | /api/comment/:id | 删除评论 |
| GET | /api/user/list | 用户列表 |
| PUT | /api/user/:id/role | 修改用户角色 |
| PUT | /api/user/:id/status | 修改用户状态 |
| GET | /api/setting | 获取系统设置 |
| PUT | /api/setting | 更新系统设置 |
| POST | /api/upload/image | 图片上传 |
